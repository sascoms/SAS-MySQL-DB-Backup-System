A) Installation - top

  1. Unzip the downloaded file
  2. Move the dbbackup folder to your web root path (or any other folder browsable on the web).
  3. Enter the application url on you favorite browser
  4. Log in using the username "admin" and password "admin" without quotes.
  5. Take a look at the Settings page and change any of them if necessary.
  6. Go to Backups page and see your databases and create/restore/delete backups

B) Application and Backup Settings - top

You can change these settings on Settings page or manually modify the definitions.php file in the root path of the application.
db host: your database server host name. ie: localhost, your.domain.com, etc.
db user: your database server user name. ie: dbuser, root, etc.
db pass: your database server user password.
db port: your database server port.
db backup folder: The folder where your backup files will be placed. You can use absolute path here. ie: /var/mybackups/mysql/, /var/www/_backups/, /home/user/db/backups/
require login for this application/system: if you will integrate this application to any of your systems, then you can disable the login/authentication of the application and use your own login system.
username: username to use for logging in the system ( default is admin)
password: password to use for logging in the system ( default is admin)
application label: this application's label and title. You can change it to anything you want. ie: My DB Backups, etc.
mysqldump path: The path for mysqldump utility.
zip path: The path for zip utility.
gzip path: The path for gzip utility.

Folder/Directory Explanations:

_backups
This folder is where your database backups will be kept.
This folder should have write permissions by your web server (Apache, etc.)
You can change this folder setting in your definitions.php file or by clicking the Settings link in the application.

controllers
This folder and the files inside it are the files that contains logic of the application.
Do not change or modify these files unless you know what you are doing.

css
This is the folder where CSS files are kept. There is only one CSS file and you can modify and change the styling of the application by modifying the only CSS file under this folder.

images
This folder holds the images used in CSS files and in the design/layout of the application.

js
This folder holds the Javascript files used in this application. Currently, there is only one file which is jQuery's main core js file.

.htaccess
Empty htaccess file (in case you need one more level for security and access.

common.php
File that contains custom php functions used in this application.

definitions.php
This file holds the settings and configuration for the application and the backup operations.
You can manually edit this file for settings and configuration of the application and backups OR you can use the Settings link on the left top corner of the pages.

index.php
This is the main index file of the application.
Do not modify or change anything unless you know what you are doing.

D) HTML Structure - top

Most of the pages in this application has an HTML structure divided into three parts. First part is in the header file and this file is common for all files.
Second part is the content of the web page and this content is generated via the php codes.
The third part is the footer and this is common for all files as well. The HTML structure is a fixed layout with one column. This column consists of one wrapper and two rows. The first row holds the title and header part of the HTML layout. And the second row is the content row where the main content is displayed.

E) CSS Files and Structure - top

I'm using one CSS file in this application. This file contains some general styling, such as anchor tag colors, font-sizes, etc. But also has styling for all elements that need special styling. Keep in mind, that these values might be overridden somewhere else in the file.

If you would like to edit the color, font, or style of any elements in the layout, you would something like the following:

			#content a {
				color: #someColor;
			}

If you would like to edit a specific section of the site, simply find the appropriate label in the CSS file, and then scroll down until you find the appropriate style that needs to be edited.

The application has a background image that is placed within the images folder. If you would like not to use it, just comment or remove the line below in the css file: 

		background-image: url(../images/body_bg.png); background-repeat: repeat-x;
